From form to live card in 5 steps

No servers, no database. You edit here, download a file, and push it to GitHub. Everything auto-saves in this browser as you type.

1
Fill in the client
Pick a client on the left (or hit New client), then edit fields. Toggle off any link a client doesn't have. Watch the phone preview update live.
2
Download the card
Hit Download index.html. That single file is the finished website for that client. Self-contained, works offline-friendly.
3
Push to GitHub
Put the file in a repo (one repo per client, or one folder each), name it index.html, then turn on GitHub Pages in the repo settings. You get a public URL.
4
Write the NTAG213 once
Use NFC Tools app → Write → URL record → paste the GitHub Pages link → hold tag to phone. Then set it read-only so nobody overwrites it.
5
Update anytime, tag never changes
Client changed their number? Edit here, re-download, re-commit to GitHub. The URL stays the same so the physical card keeps working forever.
Don't lose your clients
Data lives in this browser. Hit Backup all (.json) regularly to save a file you can Restore on any machine. That's your real database.
